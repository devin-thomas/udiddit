Udacity SQL Nanodegree

Udiddit, normalized and migration-ready.

This project redesigns the starter Udiddit schema into a relational PostgreSQL model, migrates the supplied denormalized data, and documents the work in a submission-ready report. The final solution emphasizes data integrity, clean constraints, practical indexing, and reviewer-friendly deliverables.

What Changed

  • Split users, topics, posts, comments, and votes into a normalized schema.
  • Added named constraints for uniqueness, non-empty content, referential integrity, and one-vote-per-user rules.
  • Added targeted indexes for moderation, activity, and comment-thread queries.
  • Included migration SQL and validation SQL for repeatable review.

Reviewer Assets

The repository includes the completed report, schema, migration, validation checks, and example query set used to demonstrate rubric coverage and stand-out work.

Validation Snapshot

  • Source row counts match the migrated row counts for posts, comments, topics, users, and votes.
  • Post content rules are enforced so a post contains either a URL or text, but never both.
  • Comment threading supports arbitrary depth while keeping parent comments on the same post.
  • Votes enforce one non-null user vote per post and preserve dissociated votes after user deletion.
